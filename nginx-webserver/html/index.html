Nginx Webserver

A Docker playground demonstrating nginx as a static file server. Each link below exercises a different nginx configuration feature — open your browser's developer tools (Network tab) to inspect the response headers.

Use cases

Static page /about.html

A second HTML page served from /usr/share/nginx/html. Demonstrates the catch-all location / prefix block with try_files — nginx looks for the exact file, then a directory index, then returns 404.
Health check /health

Returns {"status":"ok"} as JSON without reading any file. Uses an exact match location = /health — the fastest location type in nginx, ideal for fixed paths like health probes used by Docker and load balancers.
Cache headers /static/style.css

A CSS file served with Cache-Control: public, immutable and a 30-day expiry. Uses a regex location ~* \.(css|js|png|svg…)$ to match static asset extensions. The immutable flag tells browsers to skip revalidation entirely during the expiry window.
Custom 404 /missing

Triggers the custom 404 error page defined by error_page 404 /404.html. The error page location is marked internal so it cannot be requested directly — only nginx's own error handling can serve it.
Custom 500 /error-500

Forces a 500 Internal Server Error via return 500, which nginx intercepts with error_page 500 502 503 504 /50x.html and serves the custom server error page. The same page would appear for real upstream failures (502, 503, 504).

Response headers on this page

Security X-Content-Type-Options: nosniff — prevents MIME-type sniffing
Security X-Frame-Options: SAMEORIGIN — blocks clickjacking via iframes
Security X-XSS-Protection: 1; mode=block — legacy XSS filter for older browsers
Security Referrer-Policy: same-origin — only sends Referer on same-origin navigation
Compression Content-Encoding: gzip — HTML, CSS, JS and JSON are compressed automatically
